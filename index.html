What is CSS Grid?

CSS Grid is a two-dimensional grid layout system implemented by CSS. Grids can be used to layout page areas, grid items and user interface elements. A grid, is simply the intersecting set of horizontal and vertical lines. Once defined, either explicitly or implicitly, elements can be placed onto the grid within these columns and rows.

Grids can be defined by setting their track sizes with either fixed units, pixels, or with flexible units like percentages and the fr unit (created for CSS Grid). The fr unit represents a fraction of the available space within the grid container and one of main units we'll be utilizing in the examples related to responsive layouts.

The following examples are meant to be a guide for understanding the basic concepts of CSS Grid and their properties. While there are many resouces on CSS Grid and different ways to accomplish the same result, I have structured the code examples in a way that made the most sense to me.

Let's Set Grid Tracks

After setting the parent div to display: grid; we will now set the grid's rows and columns. To set the row heights we will use the property, grid-template-rows: ;, and to set the column widths we will use, grid-template-columns: ;.

Notice that the grid-template-rows property sets three rows, the first row at 1fr, the second row at 1fr and the third row at a fixed height of 200px. The grid-template-columns property sets three columns, the first column at 1fr, the second column at a fixed width of 150px, and the third column at 1fr.

I have also included the grid-gap property and set it to 20px of space between the rows and columns. The grid-gap: ; property is shorthand for setting the grid-column-gap: ; and grid-row-gap: ; properties. Grid-column-gap: ; sets the space between columns. Grid-row-gap: ; sets the space between rows. You can set each property individually or use the shorthand, grid-gap: ;.

Using Grid Repeat Notation

When defining the grid's rows and columns, instead of typing out, grid-template-rows: 1fr 1fr 1fr 1fr; you can instead use repeat notation, like grid-template-rows: repeat(4, 1fr).

Looking at the example code I've set, grid-template-rows: repeat(3, 250px); this is saying for the first three rows, give them a height of 250px.

For the columns, grid-template-columns: repeat(4, 150px); which defines four columns each having an equal width of 150px.

Explicit Grid VS Implicit Grid

In the previous examples we have been explicitly defining our grid. But what about content that is "outside" our defined grid? If we only define the columns, for example, two column tracks, we still get rows. Even though we did not define how the row tracks are sized they are placed implicitly. These implicit tracks are auto-sized by default and their size is determined by the content they hold. We saw this in our first example by setting the parent container to display: grid;, all the Child Items were implicitly placed into rows, auto-sized by their content.

In the code example, we have defined a two column layout, grid-template-columns: 100px, 200px;. So the first column has a width of 100px and the second column 200px. Notice how the Child Items are implicitly placed on the grid and auto-sized based on their content. Child Item 3 and Child Item 9 have additional content in them and affects their partner's height in the row.

Set Tracks in the Implicit Grid

To define the track sizes for items created in the implicit grid, use the properties, grid-auto-rows: ; and grid-auto-columns: ;.

In the css, I have set two column widths, grid-template-columns: repeat(2, 300px); resulting in two 300px columns.

Grid-auto-rows: 300px; supplies the height of 300px for grid items implicitly placed within the grid. It does not matter how many items we add to the grid, they will always be given a fixed height of 300px.

It's important to note that by using grid-auto-rows: 300px; should the content inside a grid item exceed 300px, it will be hidden from view. Since we specified a fixed unit of measurement, the grid item cannot grow beyond 300px. If we used the fr unit or specified a minmax(); property we can specify how it grows.

Minmax() Function

As we saw in the previous example, our grid items were placed implicitly with a grid-auto-row of 300px of height. What if we want to provide a minimum height to the grid items but also allow them to expand to fit their content?

The minmax() function allows us to define the minimum width or height of grid items as well as how much they can expand.

In the css, I have set three columns, each with a fluid unit of 1fr. Grid-auto-rows: minmax(100px, auto); allows the grid items to have a minimum height of 100px and allows them to grow to any size (auto) dependig on their content.

Child Item 7 has more content added to illustrate this example. It's size has expanded to fit it's content. The height of all grid items will never shrink below 100px.

Instead of using, auto, you can specify another value if you want more control over a grid item. minmax(100px, 300px); would give a minimum of 100px and a growth up to 300px. Also note that minmax() can be used anywhere can defining your grid and used to control specific grid items.

Grid Lines

Understanding Grid Lines, how to define them and being able to position items onto the grid is in my opinion the most exciting feature of CSS Grid. It allows you to quickly experiment with a responsive layout!

Remember the diagram from Grid Tracks? Well, grid lines are based on the columns and rows we define. I'll illustrate this with a diagram using grid items.

Here is a three column, three row grid. Notice that grid lines are defined at the start of a row/column, in between columns/rows, and at the end. Thus, three columns produces four grid column lines. A three column row produces four column lines.

Next we'll learn how to place items based on these grid lines.

Grid Lines Example

For this example, I have explicitly defined a three column, three row grid. Which, from the previous diagram produces four column lines and four rows lines.

In the css I have defined three columns, each with 1fr unit, taking up any available space. The rows are defined as the first row have 1fr unit of space, then the second row a fixed height of 300px, and the third row 1fr unit.

Using the class name of header, I set grid-column-start property: 1; and grid-column-end property: 4; which allows it to fill the entire first row and span across all columns.

For the aside, I set grid-column-start: 1; and grid-column-end: 2; and grid-row-start: 2; and grid-row-end: 3; placing the aside inside the first column and on the second row with a height of 300px;.

The main is placed in the second column, grid-column-start: 2;, and ends on grid-column-end: 4; allowing it to take up the remaining space. Also it's placed on the second row, grid-row-start: 2; and grid-row-end: 3; and given a height of 300px.

The footer is placed as grid-column-start: 1; and grid-column-end: 4; allowing it to stretch across all columns. The row placement is, grid-row-start: 3; and grid-row-end: 4;, placing it as the third row with a height of 1fr.

In the css comments I have include the shorthand notation for placing grid items, grid-column: 1 / 4; is the same as defining the start and end. Same with row, grid-row: 2 / 3; defines an item start at the second grid line and ending at the third grid line.

Grid Template Areas

Another method for layout is with the grid-template-areas: ; property.

Each grid item is assigned a value using the grid-area:; property. Header is, grid-area: header;, Aside is, grid-area: aside;, and so on.

In the parent div, to define the grid's layout use:
grid-template-areas:
"header header header header"
"aside main main aside-2"
"... footer footer ...";

The header item is placed on the first row, repeated four times to stretch across our four columns. Row two is defined as aside (first column), main main (second and third column), and aside-2 (fourth column). The third row is, ..., meaning an empty area, then footer footer (second and third column), and ..., another empty area.

Aligning Grid Tracks Along Columns with Justify-Content

Grid tracks are aligned to the grid container along the rows and columns. For aligning tracks along the column axis, we'll use, justify-content:;.

Supporting properties include: normal, start, end, center, stretch, space-around, space-between, space-evenly, baseline, first baseline, and last baseline.

In the css, .wrapper has justify-content: start;. Placing content to the start of container. .wrapper-2, has justify-content: end;. Places content at the end of container. .wrapper-3, has justify-content: center;. Places content in center of container. .wrapper-4, has justify-content: space-between;. Remaining space distributed between column tracks. .wrapper-5, has justify-content: space-around;. Creating space around each items. .wrapper-6, has justify-content: space-evenly;. Creating equal amounts of space between items.
